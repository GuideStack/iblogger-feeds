# Architecture Decision Records (ADRs)

This folder holds the project's Architecture Decision Records. Each ADR captures one significant technical decision: its context, the decision, and the consequences.

ADRs are authored and voted on in **IcePanel**; a copy is kept here, versioned with the code.

- How and when to write one: see the [ADR Sign-Off Procedure](../signoff-adr-procedure.md).
- Start every new ADR from the [template](adr-template.md).
- Notify reviewers using the [email templates](email-templates.md) (Tech Lead recommendation → request CTO vote → reminder → outcome).
- File naming: `adr-NNNN-short-title.md` (sequential, numbers never reused).

The Tech Lead keeps this index current.
