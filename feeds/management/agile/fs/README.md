# Functional Specs (FS)

This folder holds everything about writing Functional Specs — the document the PO writes to define *what* is being built and *why*, before any ticket exists.

- **[FS Document Procedure](fs-document-procedure.md)** — how to write an FS: who, when, the section-by-section template, and the Definition of Ready.
- **[POS Checkout & Split Payment (example)](fs-pos-example.md)** — a worked FS for a Point-of-Sale feature, showing the template filled in.

An approved FS is the input to Gate 0 of the [Ticket Sign-Off Procedure](../procedure/signoff-jira-ticket-procedure.md): the PO writes it, the PM creates tickets from it.
