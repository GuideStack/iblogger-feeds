## ១. ទិដ្ឋភាព​ក្នុង​ថ្នាក់រៀន (Classroom Dialogue)

*(ម៉ោង ៨:០០ ព្រឹក នៅសាលាវិទ្យាសាស្ត្រ​កុំ​ព្យូទ័រ លោកគ្រូយក្សដើរចូល​ក្នុង​ថ្នាក់​ដោយ​ទឹកមុខក្រហមងាំង ហើយទះតុគ្រាំង!)*

**លោកគ្រូយក្ស:** អាម៉ាប់! ក្រោកឈរភ្លាម! ហែងប្រាប់អញមើល៍ ហេតុអី​បាន​ជា​អញចុចប៊ូតុង Save ក្នុង​កម្មវិធី​ហែង​តែ​មួយវិនាទី កុំ​ព្យូទ័រ Server សាលា​របស់​អញគាំងហុយផ្សែងចេញ​មក​ក្រៅអញ្ចឹង? ហែង​សរសេរ​មេរោគចូលមែន?

**អាម៉ាប់ (ញ័រជើងទទ្រើក):** បា...បាទលោកគ្រូ! កូដ​ខ្ញុំអត់​មាន​មេរោគទេ! ខ្ញុំគ្រាន់​តែ​សរសេរ​កូដ​ឱ្យវា​លឿន! ឱ្យ​តែ​លោកគ្រូចុច Save ម្តង ខ្ញុំវាយ `new DatabaseConnection()` មួយ ដើម្បី​កុំ​ឱ្យវាច្រណែនគ្នា។ លោកគ្រូចុចរាប់សិបដង វា `new` រាប់សិបដង... វា​លឿន​មែនទែនណាគ្រូ!

**លោកគ្រូយក្ស:** យី អាកូន​នេះ! (គ្រូគប់ហ្វឺត​សរសេរ​ក្តារខៀន​ទៅ​ចំក្បាលអាម៉ាប់មួយផាំង!) លឿន​រកខ្មោចថៅ​កែ​ហែងអី! កុំ​ព្យូទ័រអញគាំង​ព្រោះ​ហែងហៅ Connection ច្រើនពេកស៊ីអស់ Memory ហ្នឹងឯង! ឯងដឹងទេថា ការ​បង្កើត Database Connection ម្តង ៗ គឺ​ស៊ីកម្លាំងម៉ាស៊ីនណាស់? គេ​ត្រូវ​ប្រើ **Singleton Pattern** វើយ! មាន​ន័យថា "ទាញយក Connection តែ​មួយគត់" មក​ប្រើរួមគ្នាមួយភូមិ!

**អាម៉ាប់ (អេះក្បាល):** អូ! អញ្ចឹងខ្ញុំគ្រាន់​តែ​បង្កើត Object មួយទុកចោល​ទៅ ហើយឱ្យ​គ្រប់​គ្នាទាញយក​ទៅ​ប្រើ​យ៉ាង​ម៉េច​ទៅ​លោកគ្រូ? ចុះបើ​មាន​ក្មេងពាល វាយកពាក្យ `new` មក​បង្កើត Connection ថ្មី​លួចលាក់នៅ​ក្រោយ​ខ្នងលោកគ្រូ តើ​លោកគ្រូដឹងអត់?

**លោកគ្រូយក្ស:** ហែងគិតថាគ្រូភ្​លើ​ដូចហែង? ដើម្បី​ការ​ពារ​កុំ​ឱ្យអាពាលឯណាវាវាយពាក្យ `new` បាន គ្រូ​ត្រូវ​ចាក់សោរទ្វារផ្ទះ! គឺ​ដាក់ `private constructor` វើយ! ពេល​ដាក់ `private` ហើយ មាន​តែ Class ហ្នឹងខ្លួនឯងទេទើបអាច `new` បាន។ បន្ទាប់​មក គ្រូ​បង្កើត​ទ្វារ V.I.P មួយឈ្មោះ `getInstance()` ជា​ប្រភេទ `static` ឱ្យពួកហែងដើរចូល​មក​សុំប្រើ!

**អាម៉ាប់ (សើចកក្អិក):** ហាហា... ចុះបើសិស្ស​ក្នុង​ថ្នាក់ ១០០នាក់ រត់សម្រុកចូលទ្វារ V.I.P ហ្នឹងព្រមគ្នា​ក្នុង​វិនាទី​តែ​មួយ (Race Condition) លោកគ្រូច្បាស់​ជា​វង្វេង ផលិត Object ១០០ ចែកឱ្យពួកខ្ញុំម្នាក់មួយ​មិន​ខាន!

**លោកគ្រូយក្ស (សម្លក់):** អេ៎! អាម៉ាប់​នេះ​ចេះគិតដល់ Multi-threading ទៀត! បើហ្វូងមនុស្សសម្រុកចូល​មក គ្រូ​ត្រូវ​ប្រើក្បាច់ **Double-Checked Locking**! គ្រូឱ្យយា​មក​ាម​ពី​រ​ជា​ន់។ ជា​ន់ទីមួយឆែកមើល បើ​ជា​ន់គ្នា គ្រូចាក់សោរទ្វារ `synchronized` មួយ​ជា​ន់ទៀត! ធានាថា​មក ១ពាន់នាក់ព្រមគ្នា ក៏ចេញ Object តែ​មួយគត់​មក​ដែរអាភ្​លើ! អង្គុយចុះ ហើយចម្លង​កូដ​នេះ​ដាក់ចូលខួរក្បាលហែង​ទៅ!

## ២. កូដ​នៅ​លើ​ក្តារខៀន (Code on the Blackboard)

```java
// សរសេរដោយលោកគ្រូយក្ស អមដោយស្នាមហ្វឺតក្រហមៗ
public class DBConnectionPool {
    
    // 1. volatile គឺដូចជាកាមេរ៉ាសុវត្ថិភាព ធានាថាគ្រប់ Thread មើលឃើញ Object ច្បាស់
    private static volatile DBConnectionPool instance;

    // 2. private constructor (ទ្វារដែកដែលអាម៉ាប់មិនអាចប្រើ new ទម្លុះបាន!)
    private DBConnectionPool() {
        System.out.println("កំពុងបង្កើត Connection ទំហំ 500MB... សុំរង់ចាំ!");
    }

    // 3. ទ្វារ V.I.P សម្រាប់ឱ្យហ្វូងមនុស្សមកសុំទាញយក
    public static DBConnectionPool getInstance() {
        // ឆែកជាន់ទី១ (ការពារកុំឱ្យរង់ចាំយូរ ពេលដែលវាមានស្រាប់ហើយ)
        if (instance == null) {
            // ចាក់សោរទ្វារ! ឱ្យចូលម្តងម្នាក់! (Synchronized Block)
            synchronized (DBConnectionPool.class) {
                // ឆែកជាន់ទី២ (ក្រែងលោមានអាណាមួយរត់ចូលមកទាន់ មុនពេលយើងចាក់សោរ)
                if (instance == null) {
                    instance = new DBConnectionPool();
                }
            }
        }
        return instance;
    }
}
```

## ៣. ពិន័យ និង​មេរៀន (Punishment & Lesson Learned)

**ពិន័យ​សម្រាប់​អាម៉ាប់:**
- ត្រូវ​ទៅ​សរសេរ​ពាក្យ `private constructor` និង `public static getInstance` ចំនួន ១០០០ ដង នៅ​លើ​ក្តារខៀន ក្រោម​ពន្លឺថ្ងៃ!

**មេរៀន​ដែល​ទទួល​បាន (Lesson Learned):**
- **បញ្ហា:** កម្មវិធី​នឹងគាំង (Out of Memory) បើ​អ្នក​បង្កើត Object ដែល​ស៊ីធនធានច្រើន ដ​ដែល ៗ ដោយ​មិន​ចាំបាច់ (ដូចជា Database Connection, File Manager, Hardware Controller)។
- **ដំណោះស្រាយ:** បិទសិទ្ធិ​មិន​ឱ្យគេប្រើ `new` ដោយ​ដាក់ Constructor ជា `private`។ រួច​ប្រើប្រាស់ `static` method ដើម្បី​ចែកចាយ Object តែ​មួយគត់​នោះ​ទៅកាន់​គ្រប់​ទិសទីទូទាំង​កម្មវិធី។
- **ចំណាំសំខាន់:** ការ​សរសេរ Singleton ត្រូវតែ​ប្រយ័ត្នរឿង Multi-threading ជា​និច្ច! បើ​មិន​ដូច្​នេះ​ទេ វានឹង​បង្កើត Object លើ​ស​ពី​មួយនៅ​ពេល​ចៃដន្យ (Race Condition)។
